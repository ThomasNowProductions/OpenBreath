## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. Please report any unacceptable behavior to the project maintainers.

## How to Contribute

There are several ways you can contribute to OpenBreath:

1. **Report bugs** - If you find a bug, please create an issue with detailed information
2. **Suggest features** - We welcome new ideas and feature requests
3. **Improve documentation** - Help us make our docs better
4. **Write code** - Fix bugs or implement new features
5. **Translate** - Help us make the app available in more languages

## Development Setup

1. Fork the repository
2. Clone your fork
3. Install Flutter SDK if you haven't already
4. Run `flutter pub get` to install dependencies
5. Run `flutter gen-l10n` to generate localization files
6. Run `flutter test` to ensure all tests pass

## Coding Standards

Please follow these coding standards when contributing:

- Follow the existing code style
- Write meaningful commit messages
- Add tests for new features
- Ensure all tests pass before submitting
- Document your code when necessary
- Keep pull requests focused on a single issue or feature

## Submitting Changes

1. Create a new branch for your changes
2. Make your changes
3. Write tests if applicable
4. Run `flutter test` to ensure all tests pass
5. Commit your changes with a clear message
6. Push to your fork
7. Create a pull request with a descriptive title and detailed description

## Reporting Issues

When reporting issues, please include:

1. A clear and descriptive title
2. Steps to reproduce the issue
3. Expected behavior
4. Actual behavior
5. Screenshots if applicable
6. Your environment (OS, Flutter version, etc.)

The more information you provide, the easier it will be for us to address the issue.
